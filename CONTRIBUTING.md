# 贡献指南

感谢你关注 PicTool Pro 项目！我们欢迎各种形式的贡献。

---

## 🎯 贡献方式

### 🐛 报告 Bug

如果你发现了 Bug，请：

1. 在 [Issues](https://github.com/LINEXIANSEN/AI-Creative-Assets-Platform/issues) 中搜索是否已有相关问题
2. 如没有，创建一个新的 Issue
3. 使用 Bug 报告模板，并提供：
   - 清晰的标题和描述
   - 复现步骤
   - 预期行为和实际行为
   - 截图或录屏（如有）
   - 浏览器和版本信息

### 💡 提出建议

我们也很乐意听到你的新功能建议！

1. 创建 Feature Request Issue
2. 描述你想要的功能
3. 解释为什么这个功能会有用
4. 提供使用场景示例

### 🔧 代码贡献

如果你想贡献代码：

1. **Fork 本仓库**
2. **克隆你的 Fork**：
   ```bash
   git clone https://github.com/YOUR_USERNAME/AI-Creative-Assets-Platform.git
   ```
3. **创建新分支**：
   ```bash
   git checkout -b feature/your-feature-name
   ```
4. **进行修改并提交**：
   ```bash
   git commit -m "Add: your feature description"
   ```
5. **推送到你的 Fork**：
   ```bash
   git push origin feature/your-feature-name
   ```
6. **创建 Pull Request**

## 📋 Pull Request 指南

### 格式要求

- 使用清晰、描述性的标题
- 详细描述你做了什么，为什么做
- 如果有相关 Issue，请关联它们（使用 `Fixes #123`）

### 代码规范

- HTML/CSS/JavaScript 使用标准格式
- 使用有意义的变量和函数名
- 添加必要的注释
- 确保代码可以正常工作

### PR 检查清单

- [ ] 我的代码可以正常运行
- [ ] 我已经测试过所有相关功能
- [ ] 我的代码没有引入新的警告或错误
- [ ] 我已经更新了相关文档（如有必要）

---

## 🧪 测试

在提交之前，请手动测试：

1. 在不同浏览器中测试（Chrome, Firefox, Safari, Edge）
2. 在桌面和移动设备上测试
3. 测试所有相关功能是否正常工作

---

## 📖 文档贡献

如果你发现了文档中的错误或想要改进文档：

1. 直接在 PR 中修改
2. 确保修改清晰、准确
3. 如有新增功能，请同步更新文档

---

## ❓ 有问题？

如果你有任何问题，欢迎：

- 给项目维护者发消息
